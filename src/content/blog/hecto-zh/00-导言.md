# hecto：用 Rust 构建你自己的文本编辑器

<img src="/images/hecto-zh/flenker-1719070814.png" alt="flenker-1719070814.png|hecto" />

**欢迎！** 这是一系列博客文章，教你如何在Rust中构建文本编辑器。

这是 <a href="http://antirez.com/news/108" class="external-link" target="_blank">反雷斯的基洛</a> 在Rust中，以及一个重混版 <a href="https://viewsourcecode.org/snaptoken/kilo/index.html" class="external-link" target="_blank">Paige Ruten 关于同样内容的 C 语言教程</a>.它大约有3000行Rust，文件和几个依赖，实现了你对最小编辑器的所有基本功能，还有语法高亮和搜索功能。

这些帖子会带你分多个小步骤构建编辑器并学习Rust。在第一步，你会添加、修改或删除几行代码。大多数步骤，你都能做到 **观察变化** 你是通过编译并立即运行程序来完成的。一旦你对Rust有了扎实的理解，你会得到下一步要解决的描述，并有机会先试用，然后再和我自己对解决方案的版本做比较。<sup><a href="#fn1" id="fnref1">[1]</a></sup>

我写这篇教程时来自工程领域的另一个角落，目的是深入了解Rust。如果你也处于类似情况，这个教程适合你。如果你已经熟悉Rust，但对文本编辑器的细节感到疑惑，这个教程同样适合你。

我会在每个步骤中详细说明，有时非常详细，而且绝对不总是与Rust相关。欢迎浏览或跳过文字，<sup><a href="#fn2" id="fnref2">[2]</a></sup> 因为这件事的主要观点是 **你将在学习Rust的同时从零开始构建文本编辑器！** 一路上学到的东西都是额外的收获，光是输入代码修改和观察结果就能学到很多东西。

参见 <a href="08-附录.md" class="internal-link" data-note-icon="" target="">附录</a> 关于教程本身的更多信息（包括卡住时该怎么办，以及哪里可以求助）。

如果你准备好开始了，那就去 <a href="01-环境搭建.md" class="internal-link" data-note-icon="" target="">第一章</a>!
