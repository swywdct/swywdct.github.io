# 附录

## 注释提交的工作原理

本教程中的大多数步骤都以带注释提交的链接形式呈现。每次提交都会显示你需要对上一步代码做的修改，以便进入当前步骤，并配有注释解释每行有趣的代码。

<a href="https://github.com/pflenker/hecto-tutorial/commit/23f1c0f6c54d794a8794fa40432c8e52103cc188" class="external-link" target="_blank">这里有一个提交的示例</a>.

每个差值包含一个修改文件的块。每个块开头都有一个头部，包含你需要编辑的文件名（“src/main.rs”）。在首部之后，显示文件内容。新增内容用绿色高亮显示，删除用红色标示。旧的行号印在左侧，新的行号印在右侧。每行有趣的代码下方都显示注释，标签如下 `(Author)` 告诉你提交的那个人（我）也说了那条评论。

在右上角，你会看到两个重要的设置：

- Split/Unified 让你从上述视图切换到左边显示旧文件、右边显示新文件的视图——然后再切换回来。
- 空白/忽略空白 选择是否显示差异中空白的更改——例如，在重新缩进代码时，变化只有每行前的空白，逻辑本身没有，所以忽略这些变化可能更合理。  
  此外，在代码块的右上角，你会发现一个 `...` 按钮。点击它可以隐藏评论，专注于更改本身。  
  例如， <a href="https://github.com/pflenker/hecto-tutorial/commit/23f1c0f6c54d794a8794fa40432c8e52103cc188?diff=unified&amp;w=0" class="external-link" target="_blank">此链接</a> 带你进入带有注释的详细版本，帮助你推理每行代码行。 <a href="https://github.com/pflenker/hecto-tutorial/commit/23f1c0f6c54d794a8794fa40432c8e52103cc188?diff=unified&amp;w=1" class="external-link" target="_blank">此链接</a> 它能带你看到更简洁的版本，无需注释或空白区域。

## 如果你陷入困境该怎么办

前几个提交是让你跟着做并复制它们的设计。

这个教程里有些代码很难准确输入，尤其是你不习惯Rust的话。当你对一条线做修改时，觉得已经完成了，但实际上错过了同一行的另一部分的一点小改动，这尤其容易出错。重要的是要慢慢来，比较差速器更换的部件 *逐角色* 用你的代码确认它们是一样的。

如果你怀疑自己犯了错，但不知道错误具体在哪里，或者可能有多早，你应该让电脑在你版本之间做个差异处理 `hecto` 以及教程版本的 `hecto` 无论你处于哪个阶段。

你需要 `git` 去做这件事。要安装git，请按以下操作操作 <a href="https://git-scm.com/book/en/v2/Getting-Started-Installing-Git" class="external-link" target="_blank">这些官方指示</a>.一旦你有了 `git` 安装后，克隆 <a href="https://github.com/pflenker/hecto-tutorial" class="external-link" target="_blank">hecto-tutorial</a> 通过运行来实现仓库 `git clone https://github.com/pflenker/hecto-tutorial.git`. `cd` 通过以下方式进入仓库 `cd hecto-tutorial`.每个提交链接的左上角都有一个提交ID，你可以查看。

要查看上面的提交，你需要输入：

``` bash
$ git checkout 2e090ab
```

文件夹中包含该步骤内容的所有文件。

你可以通过运行类似的程序来比较你的文件和这个文件夹的文件 `git diff --no-index -b ../path/to/your/main.rs main.rs` .这将显示你需要对你的修改 `main.rs` 让它看起来像仓库里的那个。该 `-b` 选项不考虑空白，所以你用的缩进样式和教程里的不一样也没关系。

过一段时间后，你会接到任务，自己尝试解决。我建议你把我最后一次提交作为你自己实验的起点。这样，当你卡住时，可以将当前状态和我下一次提交进行比较。

要比较当前代码与上述提交，你需要输入：

``` bash
$ git diff 2e090ab
```

## 改进与功能的想法

如果你想延长 `hecto` 我建议你自己尝试实际使用 `hecto` 作为你的文本编辑器。你很快就会痛苦地意识到，文本编辑器里你习惯拥有但缺失的各种功能 `hecto`.这些才是你应该尝试添加的功能。你在工作时应该用 hecto `hecto`.

如果你还在找点子，这里有一份小清单。如果你完成了其中一些，告诉我！

- 重构。我们的代码是凭空出现的，并且沿途重构过，但我不希望这个教程变成一场重构狂欢，所以代码结构没有达到理想的水平，这可能会让未来的修复和扩展更加困难。
- 测试。我们通过尝试和调试来测试代码，但随着时间推移，这变得越来越困难，而且在添加新功能时很容易丢弃或破坏旧功能。是时候加入测试了 `hecto`.
- 偏执 `clippy`.加大 `clippy`对11号的行为进行评估，看看你能否充分应对它引发的所有问题。
- 支持更多文件类型。
- 让高亮标记可配置。例如，在某些语言中，单一 `#` 用两条斜线表示一条线条。
- 允许其他选择。例如，在许多语言中，字符不会单独高亮显示，而是用单引号或双引号分隔字符串。
- 行号：在文件每行左侧显示行号。
- 自动缩进：开始新行时，将其缩进到与上一行相同的水平。
- 硬换行行：当用户准备在屏幕末端之后输入时，在文本中插入换行。尽量不要在换行词会分开的地方插入换行。
- 软包裹行：当一条线长于屏幕宽度时，使用多行显示，而不是横向滚动。
- 复制粘贴：让用户选择文本，然后在按Ctrl-C时复制选中的文本，按下Ctrl-V时粘贴复制的文本。
- 多缓冲区：允许同时打开多个文件，并有某种方式在它们之间切换。
- 非阻塞阅读： `hecto` 目前在等待用户输入时完全屏蔽，导致无法仅显示特定时间内的通知。
- 鼠标支持： `crossterm` 提供鼠标事件支持。用户可以通过点击终端来更改光标位置。

## 我写作的原因 `hecto`

回到2019年（我写这个教程的第一个版本时），我主要只熟悉JavaScript。当然，我过去有一些Java经验，但这些年来这些知识开始逐渐淡化。

与此同时，Rust 是新晋的，所以我一直在寻找学习它的方法。  
我最初的想法是玩玩一个网页服务器，但后来我想到，这和我尝试新事物时的做法很接近。然后有一天， <a href="https://viewsourcecode.org/snaptoken/kilo/index.html" class="external-link" target="_blank">Paige Ruten 很棒的教程，教你如何用 C 语言构建自己的编辑器</a> 再次登上Hacker News首页，激发了这个教程的灵感。 `kilo` 它足够复杂，足以构成挑战，多亏了Paige，它已经足够清晰，让我能够专注于学习这门语言，而不会被实现文本编辑器的细节所迷失。

- **扩展**： 如果你完成了上述某个功能添加，或扩展 `hecto` 从其他方面来说，这绝对是太棒了！只要开一个永久居民，详细评论（如果你愿意的话），告诉我你希望如何被归属。别费心把所有东西都放在一个提交里，正如我在前一步要求的那样。然后我会把你的贡献和署名添加到上方的列表中。
